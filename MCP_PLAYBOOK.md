# MCP_PLAYBOOK.md

## 목적
이 저장소의 채용 수집 기능을 **AI 에이전트(MCP 클라이언트 포함)**가 안정적으로 호출할 수 있도록 표준 실행 방법과 출력 스키마를 정의한다.

---

## 1) 실행 엔트리
모든 호출은 **프로젝트 루트**에서 아래 커맨드를 기준으로 한다.

```bash
python3 main.py --code <CODE> [--keyword <KEYWORD>]
```

- `--code`: 데이터 소스 선택
- `--keyword`: 소스별 검색어 오버라이드 (미입력 시 소스 기본값 사용)

### 리눅스/가변 경로 원칙
- 절대경로를 하드코딩하지 않는다.
- 항상 `main.py`가 있는 디렉토리로 `cd` 후 실행한다.

```bash
cd /path/to/project
python3 main.py --code all
```

---

## 2) 표준 출력 스키마
`main.py` 출력은 JSON 문자열이다.

### 단일 소스 조회 (`--code <CODE>`)
```json
{
  "기관": "기관명",
  "검색어": "검색어",
  "결과": [
    {
      "제목": "공고 제목",
      "링크": "상세 URL",
      "일자": "YYYY-MM-DD 또는 YYYY.MM.DD"
    }
  ]
}
```

### 전체 조회 (`--code all`)
```json
{
  "조회대상수": 0,
  "목록": [
    {
      "code": "소스코드",
      "기관": "기관명",
      "검색어": "검색어",
      "결과": []
    }
  ]
}
```

## 4) AI 실행 규칙 (권장)
1. 사용자 요청이 특정 기관이면 해당 `code` 1회 호출.
2. 사용자 요청이 “전체”이면 `--code all` 호출.
3. 키워드가 명시되면 반드시 `--keyword`로 전달.
4. 결과가 비어 있으면 빈 배열을 그대로 반환하고 임의 추정하지 말 것.
5. 링크 유효성 문제 제기 시 해당 소스만 재호출하여 최신 링크 재확인.

## 6) `job_portal` 추가 필드
`job_portal` 결과 항목에는 아래 필드가 추가될 수 있다.

```json
{
  "제목": "...",
  "링크": "...",
  "일자": "2026.05.22",
  "직무분류": "데이터분석가, PM ...",
  "플랫폼": "잡코리아|사람인",
  "회사": "회사명"
}
```

- `일자` 정규화 규칙:
  - `~ 05/22(금)` -> `2026.05.22` (현재 연도 기준)
  - `2026. 5. 22.` -> `2026.05.22`
  - `채용시` 등 텍스트형은 원문 유지

## 7) 장애 대응
- `pnpm` 미설치 환경: `job_portal`은 자동으로 `npm install && npm run build` 폴백.
- 특정 사이트 응답 구조 변경 시 해당 소스 `code`만 실패 가능. 다른 소스는 독립 동작.
- 네트워크 일시 장애 시 동일 명령 재시도.

---

## 8) AI용 응답 포맷 권장
AI는 사용자에게 아래 형태로 간결히 제공한다.

1. 실행한 코드와 검색어
2. 결과 건수
3. 상위 N건 (`제목`, `일자`, `링크`)
4. 결과 없음 시 명시적으로 `결과: 0건`

---

## 9) 변경 시 체크리스트
새 소스 추가/수정 시 반드시 확인:
- `main.py`의 `CRAWLER_LIST` 등록 여부
- `AGENT.md` 코드 목록 반영 여부
- 이 문서(`MCP_PLAYBOOK.md`) 코드 표 반영 여부
- `python3 -m py_compile ...` 문법 확인
- 실제 `--code <new_code>` 실행 확인
